Conceptos
Branch o Rama:
Cda vez que guarde los cambios lo llamaremos commit
Debemos estar dentro de una rama
Varios servidores como Gitlab, Gitlab, Bitbucket.
Varios Clientes como sourcetree, Git
Cada cambio que hacemos que en cada comit debemos describir hay que ser atomicos.
Los cambios que hacemos al servidor le llamamos push
Para llamar los metadatos le llamamos fetch
Cuando traes cambios y los unes con los cambios lo llamamos merge.
Cuando nos traemos todos los documentos del servidor fisicamente lo llamamos push
En la rama master se debe tener la version estable.
Enviar cambios de una rama a otra rama se llama Pull Request
Arreglar las ramas lo llamamos merge pull request.
